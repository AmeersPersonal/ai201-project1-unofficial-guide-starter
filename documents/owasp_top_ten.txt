Source 6: OWASP Top Ten Web Application Security Risks
URL: https://owasp.org/www-project-top-ten/

OWASP Top Ten is a security reference for the most critical categories of web application risk. Student side projects commonly touch authentication, user input, databases, file uploads, sessions, and third-party packages, so OWASP is useful even for small apps. The guide should remind students that a project is not "done" just because the feature works locally. They should check for broken access control, injection, cryptographic mistakes, insecure design, security misconfiguration, vulnerable dependencies, authentication failures, software and data integrity issues, missing logging, and server-side request forgery. A practical side-project checklist can translate those risks into habits: validate input, use prepared database queries or ORM safeguards, avoid exposing admin routes, store passwords with proven hashing libraries, update dependencies, and avoid overpowered API keys.
